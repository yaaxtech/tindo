# 07 — Inteligência Artificial (roadmap incremental)

## Princípios

- **IA propõe, humano dispõe.** Toda sugestão passa por aprovação antes de virar estado.
- **Transparente.** Usuário sempre vê por que a IA sugeriu X (breve explicação textual).
- **Revertível.** Toda aplicação de sugestão gera um undo.
- **Opt-in por feature.** Cada tipo de IA pode ser ligado/desligado em Configurações.
- **Barato primeiro.** Haiku pra classificação de alto volume; Sonnet pra análise; Opus só quando justificado.

## Modelos

| Uso | Modelo | Por quê |
|---|---|---|
| Classificar tarefa nova (ao criar/sync) | `claude-haiku-4-5-20251001` | Rápido, barato, suficiente |
| Análise de batch (mass update) | `claude-sonnet-4-6` | Equilíbrio custo/qualidade |
| Sugerir caminho crítico / novas tarefas | `claude-sonnet-4-6` | Raciocínio mais profundo |
| Recalibração complexa | `claude-sonnet-4-6` ou `claude-opus-4-7` | Só quando justificar |

**Prompt caching ativado** para system prompts longos (critérios de sucesso, exemplos).

## Fases

### Fase A — Calibração Inicial (trigger: usuário ativa IA)

Wizard `/calibracao` (1ª vez):

**Pergunta 1** (aberta):
> "Hoje, quais são as coisas que, se você fizesse, mais te **despreocupariam**? Liste de 3 a 7 em linguagem livre."

**Pergunta 2** (aberta):
> "Para cada uma, qual o critério de sucesso? Como você saberia que está 'resolvida'?"

**Pergunta 3** (escolha múltipla):
> "Hoje, o que mais te trava?"
> - Excesso de tarefas pequenas
> - Procrastinação em tarefas grandes
> - Falta de clareza de prioridade
> - Muitos projetos ao mesmo tempo
> - Outro: ___

**Pergunta 4** (ordenação):
> "Em ordem, priorize estes projetos (drag-and-drop)."

**Resultado**: `configuracoes.criterios_sucesso` jsonb estruturado.

**Uso**: vira parte do system prompt de toda chamada de IA.

### Fase B — Classificação Automática de Tarefa Nova

**Trigger**: tarefa criada (manualmente ou via sync Todoist).

**Input pro Claude**:
```
SYSTEM:
Você é o classificador de tarefas do TinDo. Critérios de sucesso do usuário:
{criterios_sucesso}
Projetos priorizados: {lista}
Tags disponíveis: {lista com tipo_peso e valor_peso}

USER:
Classifique esta tarefa:
- Título: {titulo}
- Descrição: {descricao}
- Projeto: {projeto}
- Data: {data}

Retorne JSON:
{
  "importancia": 0-100,
  "urgencia": 0-100,
  "facilidade": 0-100,
  "tags_sugeridas": [ids],
  "explicacao": "string curta (≤ 140 chars)"
}
```

**Output**: salvo em `sugestoes_ai` com `tipo='classificar'`, `status='pendente'`.
- Auto-aceita se `configuracoes.ai_auto_aceita_classificacao=true` (default false no início).
- Se manual: mostra modal "Sugerido pela IA: ..." ao abrir tarefa.

### Fase C — Análise em Batch

**Trigger**: `/configuracoes/ia/analisar` OU recalibração automática.

Seleciona N tarefas antigas (ex: 50 pendentes) e envia em um batch:
- Para cada tarefa: propõe nova `importancia/urgencia/facilidade` ou merge/delete/split.
- Retorna lista estruturada.
- Usuário revisa em tela tipo "inbox": aceitar todas / revisar uma a uma.

**Prompt batch** (usa caching de system prompt):
```
SYSTEM: {...critérios + contexto tags/projetos...}
USER: Para cada tarefa abaixo, proponha:
1. Ajuste de importância/urgência/facilidade
2. Se deve ser quebrada em sub-tarefas (listar sub-tarefas)
3. Se deve ser mesclada com outra (listar id)
4. Se deve ser excluída (razão)
Tarefas: [array]
Retorne array JSON.
```

### Fase D — Quebra de Tarefas

**Trigger**: botão "Quebrar" no modal de edição da tarefa, OU auto-sugestão quando facilidade ≤ 20 e sem sub-tarefas.

**Prompt**:
```
SYSTEM: {...}
USER: Quebre esta tarefa em sub-tarefas acionáveis (máx 5):
- Título: ...
- Descrição: ...
Retorne JSON:
{
  "subtarefas": [{ "titulo": "...", "facilidade_sugerida": 80 }, ...]
}
```

**UX**: mostra preview, usuário aceita / edita / descarta.

### Fase E — Sugestão de Novas Tarefas (Caminho Crítico)

**Trigger**: manual (`/sugestoes`) ou periódico (semanal).

A IA lê:
- Critérios de sucesso.
- Tarefas concluídas recentes.
- Tarefas abertas.
- Projetos ativos.

Sugere **novas tarefas** que deveriam estar na lista mas não estão, com justificativa ligada aos critérios.

**UX**: tela tipo Tinder de sugestões (swipe right = criar, left = descartar).

### Fase F — Recalibração

Três gatilhos (ver `docs/08_KPIS.md`):
1. **% alto de reavaliação**: humano muda nota da IA em >X% das tarefas.
2. **% alto de descarte de sugestões**: humano rejeita >X% das sugestões de novas tarefas.
3. **% alto de adiamento**: humano adia >X% das tarefas mostradas.

**Ação automática** (quando gatilho dispara):
- Para (1): pedir recalibração de pesos (slider 0-100 em 5 tarefas).
- Para (2): pedir recalibração de caminho crítico (o que mudou no objetivo?).
- Para (3): tela friendly de 5 tarefas adiadas com slider 0-100.

Sistema calcula correlação e propõe novos pesos (`w_urg/w_imp/w_fac`) ou ajustes nos multiplicadores de projeto/tag.

## Cache de sugestões

- `sugestoes_ai` (ver schema) armazena toda sugestão.
- Hash do input (tarefa + config.updated_at + tags/projetos.updated_at) — se bate com sugestão recente, reusa em vez de chamar API.

## Segurança

- API key (`ANTHROPIC_API_KEY`) só em server.
- Rate limit por usuário: máx 100 chamadas/dia no free; configurável.
- Logging em `logs_ai` com token count, custo estimado, latência.

## Métricas

- Custo mensal por usuário (USD).
- Taxa de aceite das sugestões (aceitas / total).
- Latência p95.
- % tarefas com IA vs sem IA aplicada.
